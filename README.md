# humanizer-zh-wanyfon

`humanizer-zh-wanyfon` 是一个面向 Codex 的中文文本改写 skill，用于识别并削弱常见的 AI 写作痕迹，让文本更自然、更像真人编辑过的中文。

它适合处理中文文案、技术教程、说明文字、文章片段、社媒内容和长文草稿。核心目标是保留原意，同时减少模板化、宣传腔、空泛铺垫和机械连接词。

## 能做什么

- 保留原意、事实、术语和基本语气
- 删除空泛开场、过度总结、宣传式表达和通用乐观结尾
- 减少"此外""然而""值得注意的是"等高频 AI 连接词
- 修正三段式列举、否定式排比、破折号过度使用、粗体滥用等格式痕迹
- 改善句子节奏，让长短句和段落结构更自然
- 处理中文技术教程常见问题，如弱限定词堆积、自问自答过密、知识树过度完整、说明书式重复
- 可按要求输出改写稿、问题说明和质量评分

## 不适合什么

- 不负责事实核查
- 不凭空补充原文没有的数据、案例、引用或结论
- 不把严肃技术文本强行改成口语化表达，除非你明确要求
- 不替代专业编辑、法律审查、医学审查或学术审稿

## 安装

在 Codex 中可以直接要求安装这个仓库：

```text
请安装这个 skill：
https://github.com/wangyuefan09/humanizer-zh-wanyfon
```

也可以手动克隆到 Codex 的 skills 目录。

Windows PowerShell:

```powershell
git clone https://github.com/wangyuefan09/humanizer-zh-wanyfon.git "$env:USERPROFILE\.codex\skills\humanizer-zh-wanyfon"
```

macOS 或 Linux:

```bash
git clone https://github.com/wangyuefan09/humanizer-zh-wanyfon.git ~/.codex/skills/humanizer-zh-wanyfon
```

安装后重启 Codex，让新的 skill 生效。

## 基本用法

在 Codex 中显式调用：

```text
请用 $humanizer-zh-wanyfon 改写下面这段文字，去掉 AI 味，保留原意：

[粘贴你的文本]
```

轻度润色：

```text
请用 $humanizer-zh-wanyfon 对下面文字做轻度润色。
要求：保留原意、事实、术语和语气；不要新增原文没有的信息；只让表达更自然。

[粘贴你的文本]
```

深度重写：

```text
请用 $humanizer-zh-wanyfon 深度重写下面文字。
要求：保留核心信息和事实，不新增数据；去掉模板化结构、宣传腔和机械连接词；让段落读起来像真人写的。

[粘贴你的文本]
```

只做审阅：

```text
请用 $humanizer-zh-wanyfon 审阅下面文字，不要直接改写。
请列出主要 AI 写作痕迹，并给出修改建议。

[粘贴你的文本]
```

输出改写稿和说明：

```text
请用 $humanizer-zh-wanyfon 改写下面文字，并输出：
1. 改写后的文本
2. 主要修改点
3. 质量评分

[粘贴你的文本]
```

## 推荐约束

为了避免改写跑偏，建议在提示词里明确这些要求：

- 保留原意、事实、术语、人名、地名、数字和结论
- 不新增原文没有的信息
- 不删除关键限制条件
- 技术文本保持准确，不强行文学化
- 学术或正式文本保持客观，不随意加入第一人称
- 如果信息不足，直接说明不足，不编造来源

## 主要识别模式

这个 skill 会重点检查以下写作痕迹：

- 过度强调意义、趋势、遗产和"重要性"
- 宣传式语言和广告腔
- 模糊归因，如"专家认为""行业报告显示"
- 三段式列举和公式化段落
- 否定式排比，如"不仅仅是...而是..."
- 过度使用连接词、破折号、粗体、表情符号
- 聊天机器人式开场和收尾
- 知识截止日期免责声明
- 中文技术教程里的过度铺垫、连续设问、知识树堆叠和重复总结

## 备注

这个 skill 的目标不是把文本改得"更华丽"，而是让文本更可信、更具体、更少模板感。最好的结果通常不是大幅扩写，而是删掉多余铺垫，保留真正有信息量的部分。
